LGU Platform Services - Project Context

1. Architecture Overview (Two Servers)
The system is deliberately divided into two main backend services to separate global system administration from localized local government unit (LGU) operations.

- Admin API (Superadmin Server):
  This server is responsible for global system orchestration. It is accessed strictly by root superadmins and platform administrators. Its primary role is to register, manage, and monitor LGU tenants. When a new LGU subscribes to the platform, the Admin API generates a secure "registrationKey" for that jurisdictional entity. It handles the suspension or activation of these tenants and acts as the ultimate source of truth for the platform's tenant registry.

- Tenant API (LGU & Citizen Server):
  This server handles the day-to-day operations of the local government units and serves the citizens. An LGU onboards onto this server using the "registrationKey" provided by the Admin API. This server maintains its own database schema to handle departments, staff members, citizen accounts, and service applications. To remain functional, the Tenant API regularly polls the Admin API to ensure its registration token is still active and valid.

2. Jurisdictional Branches (LGU Levels)
The platform is designed to handle the hierarchical structure of the Philippine local government system. The organizations (tenants) are classified into different jurisdictional levels:
- Province
- City
- Municipality
- Barangay

Each LGU operates securely within its own tenant space. Service types can be scoped to specific levels (e.g., a barangay clearance at the Barangay level, or a business permit at the Municipal level).

3. Internal Staff Operations
Staff members operate entirely within the Tenant API. 
- Onboarding: The first staff member (sysadmin) uses the registration key to initialize the organization on the Tenant API, automatically creating a default "Main Office" department.
- Departments: The LGU is subdivided into departments (mandatory, optional, or custom).
- Staff Management: The sysadmin can invite other staff members and assign them to various departments.
- Permissions: Staff are given granular permission grants (e.g., manage_hr, review_ordinance, approve_permits) which can be scoped to departments, specific application cases, or the entire organization.
- Case Processing: Staff members review, approve, or reject applications submitted by citizens. They manage internal workflows, communicate across departments, and perform backend operations that are abstracted away from the public.

4. Citizen Usage and Service Availment
Citizens interact with the Tenant API (via a citizen-facing app or portal) to avail of government services without needing to visit physical offices.
- Account Creation: Citizens create an account that is tied to the LGU they reside in or are transacting with.
- Identity Verification: The system supports different verification levels (e.g., unverified vs. id_verified) and allows integration with national IDs like the PhilSys Number.
- Availing Services: Citizens can browse available services (ServiceTypes). Depending on the service category (barangay, municipal, provincial), they can apply for clearances, permits, or certificates. Some sensitive services strictly require the citizen to have a verified identity level.
- Application Cases: To avail a service, the citizen submits an application (ApplicationCase). This case is securely routed to the appropriate assigned department within the LGU.
- Tracking: The citizen can track the real-time status of their application (e.g., "submitted", "in_review", "approved", "rejected") as it is processed by the internal LGU staff.
